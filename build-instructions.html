FitWit App - Build Instructions for APK

Project Overview

FitWit is a comprehensive health and fitness tracking application built with React Native using Expo.

This guide will help you build the Android APK file for installation on Android devices.

Step 1: Prerequisites

Before you can build the APK, you need to have the following installed:

  • Node.js (v16 or higher)
  • npm or yarn
  • Expo CLI (npm install -g eas-cli)
  • An Expo account (create one at https://expo.dev/signup)

Step 2: Clone the Repository

Clone the repository to your local machine:

git clone [repository-url]
cd fitwit

Step 3: Install Dependencies

Install all required dependencies:

npm install

Step 4: Log in to Expo

Login to your Expo account:

eas login

Step 5: Build the APK

The eas.json file is already configured in the project. To build the APK, run:

eas build -p android --profile preview

This will create an APK using the 'preview' profile defined in eas.json.

Note: The build process will happen on Expo's servers. Once completed, you will be provided with a link to download the APK file.

Step 6: Install on Android Device

Once downloaded, transfer the APK to your Android device and install it. You might need to enable "Install from Unknown Sources" in your device settings.

Additional Build Options

The project includes several build profiles in eas.json:

  • Development: eas build -p android --profile development - Creates a development build with dev tools enabled.
  • Preview: eas build -p android --profile preview - Creates an internal distribution APK.
  • Production: eas build -p android --profile production - Creates a production-ready app bundle (AAB).

Troubleshooting: If you encounter any issues during the build process, refer to the Expo Build Troubleshooting Guide.
